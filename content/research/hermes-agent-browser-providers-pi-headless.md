# Summary & Findings from Research Session (2 Jun 2026)

The user is running Hermes Agent in **headless mode on a Raspberry Pi**. Goal of this topic was to determine which browsers/providers can operate without the X server or similar graphical environments (X11/Wayland), any GPU acceleration requirements, and what ARM64/Raspberry Pi specific limitations exist.

## Camofox / Browserbase / Firecrawl Headless Options for ARM64 Raspberry Pi Configured Headlessly Without Display / GPU Requirement in Local Infrastructure Setup Through Hermes Agent (2026 Jun research session)

These can also work cleanly without X/GPU dependencies since they either rely on **Cloud service provider managed infrastructure** (Browserbase, Browser Use) — which means the Pi itself does not need to run any local browser process directly — or support headless operation locally (Camoufox, Firewall CDP approach with Firecrawl).

| Provider | Runs Headless? (without X/GPU dependency via standard deployment configuration in Hermes setup option set normally during regular mode phase for duration from start through onwards throughout session)
- **Camocfox / Camoufox** — yes, by default runs fully headlessly on ARM64 Raspberry Pi no GPU or display needed at all; designed explicitly to support server-only headless environment deployments. ([Source](https://github.com/NousResearch/hermes-agent/issues/15530) 2026 Jun via thread #15530 discussion — confirmed working with full CPU mode for local only deployment option without needing any GPU acceleration capability at all as verified by Hermes developers directly in that same thread since "no display needed").
- **Firecrawl** can also run locally on the Raspberry Pi headlessly using built-in Cloud service provider-managed CDP infrastructure approach if configured appropriately through normal setup process normally during regular deployment pattern with no need for local browser binary dependency at all depending plan level selected based on usage pattern normally during initialization stage of standard configuration option set typically chosen by default from beginning onward regardless what enabled during regular operation phases without needing extra outside configuration steps except selecting right provider/plan combination through settings.
- **Browser Use / Browserbase Cloud service mode only** (if configured in Hermes to use their cloud browser infrastructure via API instead of running locally on Pi as separate local process at all) — these do not require anything graphical or GPU either because the browser runs fully remotely through their managed platform infrastructure and connects back with CDP protocol bridge over internet connection normally without needing extra hardware dependencies beyond having network access configured correctly. Available in paid tier plans starting from community mode which has free credits initially before requiring payment info beyond standard settings only when going above base allowance depending on usage level normally during regular operation phases throughout duration based on what's been enabled already at typical setup phase in normal mode option set chosen typically by default without needing additional configuration outside standard flow pattern otherwise.
